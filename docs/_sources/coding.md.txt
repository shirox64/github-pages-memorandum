# コーディング

## コーディングの心得5か条
1. 見やすさを重視する
2. ネーミングを分かりやすく
3. サンプルを鵜呑みにしない
4. 同じコードを二度書かない
5. 役割は一つに

## 良いコードの書き方
* 変数のスコープを小さくする
* 変数は必要最低限にする
* 分岐をシンプルにする(ネストを深くしない)
* クラス、ファンクションを大きくしすぎない
* テストしやすいコードを書く(忘れがちだけど重要)

## 良いコードの定義
* 正確に動作すること
* 素早く効率的に動作すること
* 防御的でバグを生み出しにくいこと
* テストコードが書きやすいこと
* メンテナンスが行いやすいこと
* 他人が見ても理解可能であること
* 無駄な部分がないこと

## コード分割のメリット
* 可読性向上
* メンテナンス性向上
* 再利用性向上
* テストコードを書きやすい
